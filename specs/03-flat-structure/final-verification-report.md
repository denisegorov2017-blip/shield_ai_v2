## 3. Краткий обзор финальной архитектуры

Shield AI v2.0 построен на принципах **Clean Architecture** с полным разделением слоёв и зависимостей:

### Слои архитектуры:
- **Domain Layer**: Чистая бизнес-логика без зависимостей от внешних систем
  - Содержит доменные сущности (Product, Batch, ShrinkageProfile)
  - Реализует 3 стратегии усушки через Strategy Pattern
  - Интерфейсы репозиториев для инверсии зависимостей

- **Application Layer**: Сценарии использования (Use Cases)
 - CalibrateCoefficientsUseCase - Калибровка коэффициентов
 - ForecastShrinkageUseCase - Прогнозирование усушки
  - DTO для передачи данных между слоями

- **Infrastructure Layer**: Реализация внешних зависимостей
  - SQLAlchemy 2.0 ORM для работы с базой данных
  - Excel парсеры для обработки входных данных
  - DI контейнер для управления зависимостями

- **Presentation Layer**: UI и API
  - Streamlit страницы для пользовательского интерфейса
  - Страницы нумеруются от 0 до 11, каждая отвечает за отдельный аспект приложения

### Ключевые особенности:
- **Dependency Inversion**: Зависимости направлены внутрь от внешних слоёв к внутренним
- **SOLID принципы**: Все 5 принципов реализованы в архитектуре
- **Strategy Pattern**: Три стратегии усушки реализованы через паттерн стратегии
- **Строгая типизация**: MyPy Strict Mode используется во всем проекте
- **Синхронный код**: Проект использует синхронный подход без async/await
